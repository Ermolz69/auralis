feat: Implement mock job system, typed Tauri API, and decoupled adapters

This commit introduces three major architectural milestones ensuring robust, testable, and cleanly separated layers before integrating real sidecar processes (yt-dlp, ffmpeg, AI models).

1. Real Job System Implementation (crates/jobs):
- Introduced comprehensive `Job`, `JobProgress`, and `JobStatus` state machines.
- Implemented `JobManager` orchestrating jobs asynchronously using `tokio::sync::RwLock` and `tokio::spawn`.
- Designed robust cancellation with `CancellationToken` mapped to a background pipeline.

2. Frontend-Backend Typed API via Tauri:
- Enforced strict backend-orchestrated use cases (`CreateProjectUseCase` -> `ImportVideoSourceUseCase` -> `StartMockPipelineUseCase`).
- Replaced frontend stub logic with strict Tauri IPC calls residing in `shared/api/tauri`.
- Reconstructed `entities/project`, `entities/transcript`, and `entities/job` on the React side to correctly react to state invalidations.
- Handled global state invalidation via a new `ProjectContext`.
- Established event-driven synchronization (`transcript-ready`, `job-event`) reflecting live UI updates directly from Rust backend events.

3. Decoupled Mock Adapters Layer:
- Extracted hardcoded mocks from `application::usecases::test_utils` into dedicated adapter crates.
- Provided `InMemoryProjectRepository`, `InMemoryJobRepository`, and a stubbed `LocalArtifactStore` in `adapters-storage`.
- Abstracted deterministic mock inputs for `VideoSourcePort` and `SubtitleSourcePort` via `adapters-ytdlp`.
- Handled mock TTS segment generation and ASR transcripts in `adapters-model`.
- Routed media muxing operations cleanly through `adapters-ffmpeg`.
- Rewired dependency injection within `src-tauri` using exact types from external adapter crates, allowing simple swapping with real production sidecars in the future.

These structural changes solidify the application layers ensuring no domain leakage occurs during the future integration of heavy AI pipelines.
