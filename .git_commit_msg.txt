feat(application): add core application use cases and orchestrate mock pipeline

This commit introduces the foundation for the application layer by implementing the first set of core business orchestration use cases.

Key changes:
- `application` Crate Setup: Added dependencies on `domain` and `ports` crates, along with `thiserror` for error handling and `tokio`/`async-trait` for testing.
- Application Errors: Created `ApplicationError` enum to unify `ProjectNotFound`, `JobNotFound`, domain errors, and port errors. Added `thiserror::Error` derive to `DomainError`.
- Use Case Architecture: Structured the application layer with a new `usecases` module, introducing request/response DTOs for each use case to ensure backward compatibility and clean API contracts.
- CreateProjectUseCase: Implemented project creation, initializing it in Draft status and saving it to the repository.
- ImportVideoSourceUseCase: Added orchestration for validating a media source via a video source port, fetching its metadata, and importing the source into the project state.
- StartMockPipelineUseCase: Implemented a vertical scenario mock that orchestrates both a `Project` and a `Job`. It creates a dubbing job, advances it through various stages (ValidateSource, FetchMetadata, ExtractOrGenerateTranscript, SynthesizeSegments, MuxAudioTrack) while recording progress updates, and completes both the job and project.
- Test Suite: Wrote comprehensive unit tests for all use cases using in-memory implementations of the repositories and mock video source ports.
